course description

This course introduces artists and designers to computation as a creative medium. Using the web-focused coding languages of HTML, CSS, and JavaScript, we will learn basic computational methods and structures. By the end of the course, students will be able to design web pages and digital graphics from scratch and will be able to use computational thinking as a tool in their existing artistic practices.

topics covered

The course will provide an introduction to key coding concepts, including:

  • variables – name things, store values, generalize
  • conditionals – “if this, then that” branching logic
  • recursion – inputs, outputs, and iterative computational processes
  • modularity – organize complexity into reusable parts
  • full stack development – front end, back end, publishing

We will also practice habits of computational thinking (especially as related to creative processes), including:

  • debugging – persevering with systematic problem-solving techniques
  • documentation – writing clear, legible, reusable code
  • self-reference – reflecting on your artistic practice as part of your artistic practice

learning objectives

Students will be able to:

  • develop a foundation in computation thinking and digital media literacy
  • explore relationships between code, visuals, interaction, and meaning
  • gain familiarity with core programming structures for creative applications
  • create and publish webpages using HTML, CSS, and Javascript
  • practice using documentation and other online programming resources
  • use computational reasoning as a tool for both making and conceptualizing art

The following will be essential tools throughout the course:

  • p5.js Reference: p5.js documentation

There is no required textbook; we will use freely-accessible online tutorials, documentation, and community resources.

assignments

Students will complete and document a new project each week. Prompts for the projects are proposed below. These prompts may be considered as open invitations, to be taken literally or to be adjusted to your own interests. A submission is due every week, but you are encouraged to work on the prompts whenever you are inspired to do so. You may choose to respond to the same prompt multiple times.

  1. getting to know you
    Create or improve your personal website using HTML and CSS, then publish it using Github.

  2. one if by hand, two if by C++
    Define a thing to make/do. Do it entirely by hand. Then, do it again using coding tools. (Or vice versa.)

  3. strange loops
    Do a project about a coding concept using a coding structure related to the concept.

  4. if there was a problem, yo, I’ll solve it
    What project are you working on that you want to find a computational solution for? Introduce the topics/tools from this course in an existing project.

  5. final project
    Make something you love.

submissions

All projects and exercises will be published to your GitHub repository for Intro to Computation. We will set up GitHub in class, which we will use as a tool for publishing work on the web (a critical part of the web design and development process).

Once your work is published, you must submit it before class begins on the due date by posting a link to the class Are.na channel.

grading

Your grade will reflect your engagement with computational thinking through creative practice. In the words of Edgar Allen Poe (The Poetic Principle, 1850), the best works “breathe an earnestness — an evident sincerity of sentiment.” In other words, if you are genuinely interested in what you are doing, then you are on the right track.

grade breakdown:

  • weekly projects (60%): submissions that fulfill the evaluation criteria listed below
  • participation, reflection and critique (40%): thoughtful reflections on your own work and the work of your classmates; engagement during class

evaluation criteria:

  • resourcefulness: How creatively do you solve problems and find help?
  • risk-taking: Do you experiment beyond comfort zones?
  • computational integration: Are you thinking with variables, loops, conditionals, etc.?
  • documentation: Can others understand what you made and why?
  • collaborative learning: Do you help others and seek help when needed?

what grades mean:

  • A: consistent experimental making, takes creative risks, fluent computational thinking, frequently contributes to class discussions, submits work on time
  • B: regular making with some computational concepts integrated, contributes to class discussions, submits work on time
  • C: completes basic requirements, shows effort but limited integration of concepts, minimal contributions to discussions
  • Below: inconsistent work, missing or late assignments, minimal computational thinking, not engaged in discussions

classroom expectations

participation

Please arrive on time to every class and participate actively in discussions and other group activities. I (AnneMarie) want you to want to be here!

We will follow RISD's attendance policy (multiple unexcused absences will result in a failing grade). If you need to be absent or participate remotely, please email AnneMarie as soon as possible. Open communication is key.


growth mindset

The coding process is inherently experimental. Expect to be confused, surprised, and occasionally frustrated - this is where learning happens. In this class, we will value curiosity, resourcefulness, helping each other, and taking creative risks over getting things "right." We all are always learning.


respect

Software development and other STEM fields have historically excluded disadvantaged groups ( despite their essential contributions to the field ). In this class, we must commit to creating an inclusive space where different backgrounds, ideas, and approaches to computational thinking strengthen our collective learning.


a note on AI

Assignments are designed to help you develop computational skills, not to find a “right” answer. It is up to you, the student, to make sure that you are thinking deeply about whatever you want to learn.

While the use of AI tools is not forbidden in this course, you are encouraged to limit your use of these tools. Always attempt to solve a problem on your own first, before consulting AI. Per RISD’s Academic code of Conduct (linked below) you must always acknowledge your use of AI tools with a formal citation, and reflect on the impact of AI on a) your work, and b) your understanding of your work.
